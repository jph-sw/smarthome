## Wo?

Fumadocs findet man in der Repo [Smarthome](https://github.com/jph-sw/smarthome)

## Install

Fumadocs benötigt Nodejs und npm oder Bun damit es man es Lokal testen kann.
Um an der Doku zuarbeiten muss die repo mit auf den Lokalen rechner gecloned werden.

```
git clone https://github.com/jph-sw/smarthome.git
```

**Bei Änderungen**

Erst
```
git pull
```
<Callout title="Wenn eigene Änderungen vorliegen erst">
    ```
    git stash
```
</Callout>

Dann einmal `npm install` oder `bun install` ausführen.


## Use

Um in Fumadocs neue Ordner zu erstellen muss unter content/docs einen ordners erstellen.
Als Beispiel: (beispiel-ordner)

Um eine Seite in dem ordner zuerstellen muss im Ordner eine file.mdx erstellt werden.

Jede Seite muss mit folgendem beginnen.

```
---
title:
description:
---
```

<Callout title="Wichtig">
  Wenn man einen neuen Ordner erstellt, **muss** in der übergeordneten
  `meta.json` der Ordnername hinzugefügt werden.
</Callout>

Um den live zustand der Doku zusehen kann die Doku lokal gehostet werden unter [http://localhost:3000/](http://localhost:3000).
Mit dem Befehl npm oder bun wird dann die Lokale Doku gestartet.

```
npm run dev
bun run dev
```
