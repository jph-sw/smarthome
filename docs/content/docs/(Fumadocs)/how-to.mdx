## Install

Fumadocs benötigt **Node.js** und **npm** oder **Bun**, damit es lokal getestet werden kann.
Um an der Dokumentation zu arbeiten, muss das Repository auf den lokalen Rechner geklont werden:

```sh
git clone https://github.com/jph-sw/smarthome.git
```

## Änderungen einpflegen

Falls bereits eigene Änderungen vorliegen, sollten diese zunächst zwischengespeichert werden:

```sh
git stash
```

Anschließend das neueste Update vom Repository holen:

```sh
git pull
```

Gespeicherte Änderungen wiederherstellen:

```sh
git stash pop
```

Danach die Änderungen zum Commit vorbereiten:

```sh
git add .
```

Commit mit einer aussagekräftigen Nachricht erstellen:

```sh
git commit -m "edit: Der Pfad wo man dran arbeitet.mdx Und kurze Nachricht"
```

Die Änderungen ins entfernte Repository hochladen:

```sh
git push
```

### Nach jedem `git pull`

Nach dem Aktualisieren des Repositories sollte sichergestellt werden, dass alle Abhängigkeiten installiert sind:

```sh
npm install  # oder bun install
```

## Nutzung

Um in Fumadocs neue Ordner zu erstellen, muss unter `content/docs` ein neuer Ordner angelegt werden.
**Beispiel:** `beispiel-ordner`

Innerhalb des Ordners wird eine neue `.mdx`-Datei erstellt.

Jede neue Seite beginnt mit folgendem Header:

```mdx
---
title: "Titel der Seite"
description: "Kurze Beschreibung"
---
```

### Wichtiger Hinweis

Wenn ein neuer Ordner erstellt wird, **muss** der Ordnername in der übergeordneten `meta.json` hinzugefügt werden.
