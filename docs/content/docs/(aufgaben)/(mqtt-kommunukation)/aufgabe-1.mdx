# Grundlegende MQTT-Kommunikation Aufgabe 1

## Test Message
Test Message ist ein `node injection` und als String eingestellt mit der Nachricht "Test Message".

![MQTT Kommunikation](/Aufgabe1/string-test-message.jpeg)

## Send Test Message
Send Test Message ist ein MQTT-Out-Node, welcher als Server `localhost:1883` mit dem Admin-Account konfiguriert ist, da Schreibrechte benötigt werden. Über den Edit-Button können die Eigenschaften angepasst werden.

![MQTT Kommunikation](/Aufgabe1/mqtt-output-test-message.jpeg)

## Verbindungseinstellungen
Nach dem Öffnen der Verbindungseinstellungen kann der Server konfiguriert werden. Hier wird `localhost` mit dem entsprechenden Port angegeben.

![MQTT Kommunikation](/Aufgabe1/bearbeitung-button.jpeg)

## Sicherheitseinstellungen
Im Tab "Sicherheit" wird ein Benutzer konfiguriert. Hier wird der Admin-Account genutzt, um vollen Zugriff zu gewährleisten und spätere Probleme zu vermeiden.

![MQTT Kommunikation](/Aufgabe1/sicherheit-rechte-admin.jpeg)

## Verbindung der Test Messages
Test Message und Send Test Message wurden verbunden. Dadurch wird bei einem Button-Klick die Nachricht an den Server gesendet.

![MQTT Kommunikation](/Aufgabe1/verbindung-der-test-message.jpeg)

## Listen to Test-Topic
Listen to Test-Topic ist ein MQTT-In-Node, der ebenfalls mit `localhost:1883` und dem Admin-Account konfiguriert ist. Dadurch kann der Server empfangene Nachrichten verarbeiten. Die Konfiguration entspricht der des Outputs, allerdings handelt es sich hierbei um einen Input.

![MQTT Kommunikation](/Aufgabe1/mqtt-input.jpeg)

## Debug Node
Ein Debug-Node dient zur Ausgabe der empfangenen Nachricht und funktioniert ähnlich wie `print()` oder `echo`. Der Debug kann über einen Button aktiviert oder deaktiviert werden.

![MQTT Kommunikation](/Aufgabe1/debug-button.jpeg)

## Verbindung zwischen Input und Debug
Hier wird der Input vom Listen-to-Test-Topic-Node an den Debug-Node weitergeleitet.

![MQTT Kommunikation](/Aufgabe1/mqtt-input-and-debug.jpeg)

## Funktionsweise
Drückt man den Button bei Test Message, wird eine vordefinierte Nachricht an `localhost` gesendet. In diesem Beispiel handelt es sich um die Test Message.

![MQTT Kommunikation](/Aufgabe1/function.jpeg)

## Debugger-Ausgabe
Beim Öffnen des Debuggers und Drücken des Buttons wird die gesendete Nachricht ausgegeben.

![MQTT Kommunikation](/Aufgabe1/debug-message.jpeg)
