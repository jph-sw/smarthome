# Grundlegende MQTT-Kommunikation Aufgabe

<div class="border-2 border-white p-4 mt-8">
Test Message: ist ein `node injection`, und ist als string eingestellt
Mit der Nachricht "Test Message".

![MQTT Kommunikation](/Aufgabe1//string-test-message.jpeg)
</div>

<div class="border-2 border-white p-4 mt-8">
Send Test Message: ist ein node mqtt out, und ist als Server localhost:1883 gestellt, mit den Admin account.
Da es schreib Berechtigung braucht. Durch den Edit Button kommen wir zu den Eigenschaften.

![MQTT Kommunikation](/Aufgabe1//mqtt-output-test-message.jpeg)
</div>

<div class="border-2 border-white p-4 mt-8">
Danach befindet man sich bei der Verbindung, und kann den Server einstellen.
Da haben wir wieder localhost und den Port angegeben.

![MQTT Kommunikation](/Aufgabe1//bearbeitung-button.jpeg)
</div>

<div class="border-2 border-white p-4 mt-8">
Danach sind wir in Sicherheit reingegangen, da wir ein Benutzer einstellen müssen.
Wir haben den admin genommen, da er auf alles zugriff hat, und im späteren keine Probleme sich ergeben.

![MQTT Kommunikation](/Aufgabe1//sicherheit-rechte-admin.jpeg)
</div>

<div class="border-2 border-white p-4 mt-8">
Test Message und Send test Message haben wir verbunden.
Das bedeutet, wenn wir auf den Button drücken, wird die Nachricht an den Server gesendet.

![MQTT Kommunikation](/Aufgabe1/verbindung-der-test-message.jpeg)
</div>

<div class="border-2 border-white p-4 mt-8">
Listen to test-topic: ist ein node mqtt in, und ist als Server localhost:1883 gestellt, mit den Admin account.
Damit wir ein input bekommen, also eine Nachricht zum Server.
Das ganze ist genauso eingestellt wie der Output, nur das es ein Input ist.

![MQTT Kommunikation](/Aufgabe1/mqtt-input.jpeg)
</div>

<div class="border-2 border-white p-4 mt-8">
debug 1: ist ein debug node, der dazu dient um die Nachricht auszugeben.
Funktioniert in dem falle wie ein print() oder echo.
Und da ist ein Button, also man kan den debug aktivieren oder deaktivieren.

![MQTT Kommunikation](/Aufgabe1/debug-button.jpeg)
</div>

<div class="border-2 border-white gap-2 p-4 mt-8">
Hier haben wir die beiden verbunden, das der Input vom Listen to test-topic, an den Debug weitergeleitet wird.

![MQTT Kommunikation](/Aufgabe1/mqtt-input-and-debug.jpeg)
</div>

<div class="border-2 border-white gap-2 p-4 mt-8">
Was da jetzt genau passiert ist, das man bei Test Message einen Button hat, wenn man den drückt.
Sendet er an den locahost eine Nachricht, die wir konfiguriert haben. Im unseren falle ist es einfach eine Test Message.

![MQTT Kommunikation](/Aufgabe1/function.jpeg)
</div>

<div class="border-2 border-white gap-2 p-4 mt-8">
Und wenn man jetzt den Debugger öffnet, und auf den Button drückt, wird die Nachricht ausgegeben.

![MQTT Kommunikation](/Aufgabe1/debug-message.jpeg)
</div>
